建立腳本與環境變數設定¶

1. 建立新腳本（初始參數）¶

當你在 Configure（配方管理）中按下 New 建立新的測試腳本流程時，系統會先開啟「初始參數編輯」視窗，用來建立該腳本流程的基本識別資訊（Project / Mode / Station / Fixture / Version 等）。

2. 欄位說明¶

欄位 說明 範例
Project 專案名稱，通常對應產品專案 VO0301
Mode 腳本使用模式 PROD / ENG / GRR
Station 站別（測試站） PCBA01
Fixture 治具編號（同站多台治具用 J01/J02...） J01
Version 腳本版本，用於辨識腳本更新 V0.0.0.1
Memo 備註（選填） 例如：注意事項、變更點

3. 版本與變數取用（% 變數）¶

腳本流程建立後，系統在實際執行時會依據流程設定與環境變數，自動建立「可被流程中引用」的變數（常用於檔名、路徑、通訊指令、記錄內容等）。

常見寫法：%變數名稱%，例如：

  • %ProjectName%
  • %StationName%
  • %FixtureName%
  • %Version%
  • %ProductSN%
