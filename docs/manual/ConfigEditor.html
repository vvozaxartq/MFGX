    環境變數設定¶

    1 新增 DUT 類物件¶

    在 DUT 類裝置的屬性清單中，找到 EnvirVariable 欄位，點右側的 「…」 按鈕，即會直接開啟「編輯變數」視窗（多行編輯介面）。

    開啟後會看到類似 INI 的多段落內容（以 [區塊] 分段），每行以 Key=Value 設定：

    2 格式規則（INI 風格）¶

    • 區塊：以 [區塊名稱] 表示，例如 [UI]、[Path]。
    • 鍵值：每行一筆設定，格式為 Key=Value。
    • 替換變數：在字串中可使用 %變數名% 形式（例如 %ProjectName%）在執行時自動替換。
    • 路徑建議：Windows 路徑請避免非法字元 \ / : * ? " < > |（尤其是檔名模板）。

    3 常用區塊與鍵值說明¶

    參數（Key） 用途（詳細說明）
    ProductSN 定義序號（SN）或條碼格式規則，可在 [SerialNumber] 下自定義多組序號格式，並依設定在主 UI 視窗中生成序號輸入 TextBox。當掃碼或手動輸入 SN 時會先比對 Regex，格式不符合時可阻擋流程進入或提示錯誤。

    例：^[A-Z0-9]{12}$（12 碼大寫英數）
    TreeShow 控制主畫面左側 TreeView（測試流程樹狀結構）是否顯示。若操作員只需按 Start 測試而不需看到流程細節，可關閉以簡化畫面並降低誤操作機率。
    TitleBarcodeShow 控制標題列（Title Bar）是否顯示 Barcode/SN 的資訊區塊。若現場已用其它方式顯示 SN，或需縮減標題列資訊，可關閉此區以提升畫面可用空間。

    常見值：0/1
    SignalLightShow 控制是否顯示三色燈（Signal Light）或狀態燈 UI 元件。常用於現場快速辨識站台狀態（RUN / PASS / FAIL / ALARM）。若現場未使用燈號，可關閉。

    常見值：0/1
    TitleShow 控制主畫面是否顯示整個標題列區塊（包含版本、Recipe 名稱等資訊）。若畫面尺寸較小或希望操作畫面更簡潔，可關閉標題列。

    常見值：0/1
    RunMode 設定系統執行模式（依專案定義），會影響流程啟動方式、按鈕可用性與運作行為。

    例：0=由 SN TextBox 觸發；1=Start 按鈕觸發且可循環測試；2=圓盤模式（示意）
    BackupFolder 設定備份路徑（可帶替換變數）。測試完成後 Log/CSV/Image 等檔案會依設定備份到此資料夾。多工位/多治具環境建議包含 Project/Station/Fixture 等資訊避免混檔。通常會同時設定 Local 與 Server 各一組。

    例：C:\Camera\%ProjectName%\%StationName%\%FixtureName%
    LogFileName 設定 Log / CSV / 影像等輸出檔案的命名模板。通常加入 SN、Project、Station、治具、時間戳等以確保唯一性並方便追溯。

    例：%ProductSN%_%ProjectName%_%StationName%_%FixtureName%_%FixturePart%_%NowTimeHHmmss%

    4 範例：模板¶

    
[SerialNumber]
ProductSN=[A-Z0-9]{12}
MAC=[A-Z0-9]{2}:[A-Z0-9]{2}:[A-Z0-9]{2}:[A-Z0-9]{2}:[A-Z0-9]{2}:[A-Z0-9]{2}

[UI]
TreeShow=1
TitleBarcodeShow=0
SignalLightShow=0
TitleShow=1
RunMode=0

[Path]
BackupFolder=C:\Camera\%ProjectName%\%StationName%\%FixtureName%
BackupFolder2=Z:\Camera\%ProjectName%\%StationName%\%FixtureName%

[Common]
LogFileName=%ProductSN%_%ProjectName%_%StationName%_%FixtureName%_%FixturePart%_%NowTimeHHmmss%
FixturePort=J01
LogUseItemCode=0
